# Excavator Part Detector — YOLOv8 → Meta XR / Unity

Trains a real-time object detector to identify **5 parts of an excavator** from images or live video, then exports the model for use inside a **Meta Quest VR headset via Unity**.

---

## What it detects

| ID | Class | Description |
|----|-------|-------------|
| 0 | Boom | Upper arm of the excavator |
| 1 | Bucket | Scooping attachment at the end |
| 2 | Cab | Operator cabin |
| 3 | Stick | Forearm / dipper arm |
| 4 | Wheels | Undercarriage / tracks |

## How the pipeline works

### Step 1 — Prepare the dataset

```bash
python prepare_dataset.py
```

- Finds all images in `images/` that have a matching label file in `labels/`
- Randomly splits them 80% train / 20% val (seed fixed for reproducibility)
- Copies files into `train/` and `val/` subdirectories
- Writes `dataset.yaml` with the correct paths and class list

Labels are in [YOLO format](https://docs.ultralytics.com/datasets/detect/): each `.txt` file contains one row per object:
```
class_id  center_x  center_y  width  height
```
All values are normalized 0–1 relative to image dimensions. Labels were created using [Label Studio](https://labelstud.io/).

---

### Step 2 — Train

```bash
yolo detect train data=dataset.yaml model=yolov8n.pt epochs=100 imgsz=640 batch=8 project=runs name=excavator_v1
```

- Fine-tunes **YOLOv8 nano** (`yolov8n.pt`) on the excavator dataset
- Outputs weights, metrics, and visualizations to `runs/excavator_v1/`
- Best weights are saved to `runs/excavator_v1/weights/best.pt`
- To train an improved version, change `name=excavator_v2` (or v3, etc.) — `convert_for_unity.py` will automatically pick up the latest version

Key training metrics tracked per epoch (in `results.csv`):
- `metrics/mAP50(B)` — mean average precision at IoU 0.5 (main accuracy metric)
- `metrics/mAP50-95(B)` — stricter mAP averaged across IoU thresholds
- `train/box_loss`, `train/cls_loss` — bounding box and classification losses

---

### Step 3 — Export for Unity / Meta XR

```bash
python convert_for_unity.py
```

Auto-detects the latest `excavator_vN` training run and:

1. Exports `best.pt` → standard ONNX (raw output shape: `[1, 9, 8400]`)
2. Rewrites the ONNX graph to split that into 3 separate outputs that the **Meta XR Building Blocks** Unity SDK expects:

| Output | Shape | Type | Content |
|--------|-------|------|---------|
| `output_0` | `[8400, 4]` | float | Bounding boxes |
| `output_1` | `[8400]` | int64 | Class index per detection |
| `output_2` | `[8400]` | float | Confidence score per detection |

Saves the result as `best_metaxr_vN.onnx` in the project root.

**To use in Unity:** drag `best_metaxr_vN.onnx` into your Assets folder and configure the class labels as:
```
0: Boom  1: Bucket  2: Cab  3: Stick  4: Wheels
```

---

### Optional — Test on webcam

```bash
python test_webcam.py
```

Runs live inference from your webcam using the trained model, with bounding boxes drawn in real time. Press `Q` to quit.

> By default this loads `runs/excavator_v1/weights/best.onnx`. Edit line 6 of `test_webcam.py` to point at a different run.

## Dataset

Images are sourced from the [JCB Image Dataset](https://www.kaggle.com/datasets/dataclusterlabs/jcb-image-dataset) by DataCluster Labs on Kaggle. A subset of images was selected and hand-annotated using [Human Signal](https://app.humansignal.com/) (the hosted version of Label Studio).

Because the images are subject to Kaggle's dataset license, they are **not included in this repository**. To reproduce the training:

1. Download images of your choice from the Kaggle dataset linked above
2. Place them in `images/`
3. Annotate them in [Human Signal](https://app.humansignal.com/) using the classes `Boom`, `Bucket`, `Cab`, `Stick`, `Wheels` and export in YOLO format to `labels/`
4. Then follow the pipeline steps below

## Requirements

```bash
pip install ultralytics opencv-python onnx
```

Python 3.8+ recommended.

`yolov8n.pt` is not included in this repo. It will be downloaded automatically the first time you run the training command, or you can download it manually from [Ultralytics](https://github.com/ultralytics/assets/releases).
